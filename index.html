Best Practices

Follow these guidelines to ensure optimal user experience and performance.

  • Keep titles concise: Use short, descriptive titles that clearly identify the application or current context.
  • Use consistent branding: Include your application icon and maintain consistent color schemes across all windows.
  • Provide visual feedback: Ensure window controls have clear hover and active states for better user interaction.
  • Handle window states: Update the title bar appearance based on window focus, maximized state, and other conditions.
  • Consider dark mode: Ensure your title bar works well in both light and dark themes.
  • Test across platforms: Verify title bar behavior on Windows, macOS, and Linux if supporting multiple platforms.

Warning: Avoid making the title bar too tall as it reduces available content space. The recommended height is 32px for optimal balance.

Accessibility Guidelines

Ensure your title bar is accessible to all users, including those using assistive technologies.

  • Keyboard Navigation: All window controls must be accessible via keyboard (Tab, Enter, Space).
    button.title-bar-btn:focus { outline: 2px solid var(--primary-color); outline-offset: 2px; }
  • ARIA Labels: Provide descriptive labels for screen readers.
    <button class="title-bar-btn" aria-label="Minimize window" title="Minimize"> <i class="fas fa-window-minimize"></i> </button>
  • Color Contrast: Ensure sufficient contrast ratio (4.5:1 minimum) between text and background colors.
  • Focus Indicators: Provide clear visual focus indicators for keyboard navigation.
  • Screen Reader Support: Use semantic HTML and proper ARIA attributes.
    <div class="title-bar" role="banner" aria-label="Application title bar"> <h1 class="title-bar-text">App Name</h1> </div>
  • High Contrast Mode: Test with Windows High Contrast mode and ensure all elements remain visible.

WCAG Compliance: This component follows WCAG 2.1 AA guidelines when implemented with proper ARIA attributes and color contrast ratios.

Versatility: This sidebar adapts to any application type - media players, code editors, business tools, system utilities, and more. Simply change icons, colors, and content to match your use case.

Best Practices

Guidelines for creating effective and user-friendly sidebar navigation.

  • Logical grouping: Organize navigation items into logical categories that match user mental models and workflows.
  • Clear visual hierarchy: Use consistent spacing, typography, and visual cues to establish clear information hierarchy.
  • Meaningful icons: Choose universally understood icons that clearly represent their function or content area.
  • Responsive design: Ensure sidebar works well on different screen sizes, consider collapsible options for mobile.
  • Badge usage: Use notification badges sparingly and only for actionable items that require user attention.

Accessibility Guidelines

Ensure your sidebar navigation is accessible to all users.

  • Semantic Navigation: Use proper HTML5 nav elements and ARIA landmarks.
    <nav class="sidebar" role="navigation" aria-label="Main navigation"> <ul class="tab-list" role="menubar"> <li class="tab-item" role="menuitem">...</li> </ul> </nav>
  • Keyboard Navigation: Support Tab, Enter, Space, and Arrow keys for navigation.
  • Screen Reader Support: Provide descriptive labels and state information.
    <button aria-expanded="false" aria-controls="submenu-1"> Database </button>
  • Focus Management: Maintain logical focus order and visible focus indicators.

// Modal JavaScript Functions
function showModal(modalId) {
    const modal = document.getElementById(modalId);
    if (modal) {
        modal.style.display = 'flex';
        document.body.style.overflow = 'hidden';

        // Close on backdrop click
        modal.addEventListener('click', function(e) {
            if (e.target === modal) {
                closeModal(modalId);
            }
        });
    }
}

function closeModal(modalId) {
    const modal = document.getElementById(modalId);
    if (modal) {
        modal.style.display = 'none';
        document.body.style.overflow = '';
    }
}

// Close modal with Escape key
document.addEventListener('keydown', function(e) {
    if (e.key === 'Escape') {
        // Close all open modals
        const openModals = document.querySelectorAll('.modal[style*="flex"]');
        openModals.forEach(modal => {
            modal.style.display = 'none';
        });
        document.body.style.overflow = '';
    }
});
